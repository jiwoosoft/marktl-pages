설치된 프로그램을 다른 드라이브로 옮기는 법

C드라이브 용량이 부족할 때 프로그램을 지우고 다시 설치하지 않아도 됩니다. FolderMove와 심볼릭 링크를 이용하면 실제 데이터는 다른 드라이브로 옮기고, 윈도우에는 원래 위치에 있는 것처럼 보이게 만들 수 있습니다.

Abstract

이 글은 윈도우 PC에서 C드라이브 용량 부족을 해결하기 위해 설치된 프로그램 폴더를 다른 드라이브로 옮기는 방법을 설명합니다. 핵심은 실제 파일은 D드라이브 같은 보조 저장소로 이동하고, 원래 C드라이브 위치에는 심볼릭 링크를 남겨 윈도우와 프로그램이 기존 경로를 계속 사용하는 것처럼 만드는 방식입니다.

핵심 발견

재설치 없이 이동 가능

프로그램 폴더를 옮긴 뒤 심볼릭 링크를 만들면 레지스트리나 바로가기 경로를 대거 수정하지 않아도 됩니다.

FolderMove는 절차를 단순화

명령 프롬프트에서 직접 mklink를 입력하지 않고도 폴더 이동과 링크 생성을 한 번에 처리합니다.

검증이 필수

링크 표시, 대상 경로, 실제 저장 위치, 프로그램 실행 상태를 확인해야 데이터 손상이나 경로 오류를 줄일 수 있습니다.

복구 절차도 준비해야 함

보조 드라이브 교체, 속도 저하, 오류 발생에 대비해 심볼릭 링크 삭제와 원본 위치 복사 절차를 알아두는 것이 좋습니다.

“이 방법의 핵심은 파일을 숨기는 것이 아니라, 윈도우가 익숙한 경로를 계속 바라보게 만드는 데 있습니다.”

C드라이브 용량 부족 해결의 핵심

윈도우와 대부분의 프로그램은 기본 설치 위치로 C드라이브를 사용합니다. 게임, 영상 편집 프로그램, 개발 도구처럼 용량이 큰 소프트웨어를 몇 개만 설치해도 C드라이브의 여유 공간은 빠르게 줄어듭니다.

심볼릭 링크는 이 문제를 우회하는 실용적인 장치입니다. 실제 데이터는 용량이 넉넉한 다른 드라이브로 옮기고, 원래 위치에는 “실제 파일은 이쪽에 있다”는 연결 경로를 남깁니다. 운영체제와 프로그램은 기존 C드라이브 경로를 그대로 호출하지만, 파일 접근은 보조 드라이브의 실제 폴더로 이어집니다.

심볼릭 링크 기반 이동 구조

원래 위치
C:\Program Files\App
→
심볼릭 링크
경로 연결
→
실제 데이터
D:\Backup\App

프로그램 재설치가 필요 없는 이유

  • 용량을 많이 차지하는 프로그램 폴더를 통째로 보조 드라이브로 이동합니다.
  • 원래 설치 경로에는 심볼릭 링크를 생성합니다.
  • 윈도우와 프로그램은 기존 C드라이브 경로를 계속 사용하는 것으로 인식합니다.
  • 사용자 설정, 실행 경로, 내부 참조를 크게 건드리지 않아 오류 가능성을 줄입니다.
주의할 점. 모든 프로그램이 폴더 단위 이동에 완벽히 적합한 것은 아닙니다. 드라이버, 보안 프로그램, 백그라운드 서비스가 깊게 연결된 소프트웨어는 이동 전에 백업과 복구 계획을 먼저 세우는 것이 안전합니다.

FolderMove로 설치된 프로그램 옮기기

FolderMove는 심볼릭 링크를 직접 만드는 과정을 단순화한 포터블 도구입니다. 사용자는 원본 폴더와 이동할 대상 폴더만 지정하면 됩니다. 이후 도구가 폴더 이동과 심볼릭 링크 생성을 이어서 처리합니다.

항목 의미 확인 포인트
Move From 현재 프로그램이 설치된 원본 경로 바탕화면 바로가기의 “파일 위치 열기”로 확인합니다.
Move To 실제 데이터가 옮겨질 새 위치 용량이 충분하고 항상 연결되는 드라이브를 선택합니다.
Move and set Symbolic link 폴더 이동과 링크 생성을 실행하는 버튼 두 경로가 정확한지 확인한 뒤 실행합니다.

1단계. FolderMove 다운로드와 실행

  1. FolderMove 공식 홈페이지에 접속합니다.
  2. 무료 다운로드 버튼을 눌러 다운로드 페이지로 이동합니다.
  3. 공유 사이트에서 무료 다운로드를 선택합니다.
  4. 보안 확인을 완료한 뒤 다운로드 링크를 생성합니다.
  5. FolderMove.exe 파일을 내려받고 실행합니다.

2단계. 원본 프로그램 경로 확인

  1. 바탕화면이나 시작 메뉴에서 옮길 프로그램의 바로가기를 찾습니다.
  2. 아이콘을 마우스 오른쪽 버튼으로 클릭하고 “파일 위치 열기”를 선택합니다.
  3. 파일 탐색기 주소 표시줄을 클릭해 전체 경로를 선택합니다.
  4. Ctrl + C로 원본 폴더 경로를 복사합니다.

3단계. 새 저장 위치 지정과 링크 생성

  1. FolderMove의 Move From 칸에 복사한 원본 경로를 붙여넣습니다.
  2. Move To 옆의 Browse 버튼을 눌러 보조 드라이브의 대상 폴더를 선택합니다.
  3. 원본 경로와 대상 경로가 정확한지 다시 확인합니다.
  4. Move and set Symbolic link 버튼을 눌러 이동을 시작합니다.
  5. 완료 메시지가 나타나면 작업을 종료합니다.
예시 경로 원본 위치: C:\Program Files\OBS Studio 이동 위치: D:\Programs\OBS Studio 결과: C드라이브에는 링크가 남고, 실제 파일은 D드라이브에 저장됩니다.

이동 결과 검증과 실행 테스트

폴더 이동이 끝났다면 용량이 늘어난 것만 보고 끝내면 안 됩니다. 심볼릭 링크가 제대로 만들어졌는지, 실제 데이터가 보조 드라이브에 있는지, 프로그램이 이전과 동일하게 실행되는지 확인해야 합니다.

검증 체크리스트

  • 원래 C드라이브 위치의 폴더 아이콘에 바로가기 또는 링크 표시가 있는지 확인합니다.
  • 속성 화면에서 대상 경로가 새 저장 위치를 가리키는지 확인합니다.
  • D드라이브 같은 대상 위치에 실제 파일이 온전히 옮겨졌는지 확인합니다.
  • 내 PC에서 C드라이브 여유 공간이 늘어났는지 확인합니다.
  • 기존 바로가기나 시작 메뉴 아이콘으로 프로그램을 실행합니다.
  • 사용자 설정, 프로필, 플러그인, 작업 내역이 유지되는지 확인합니다.
  • 실행 중 “파일을 찾을 수 없습니다” 같은 경로 오류가 없는지 살펴봅니다.
증상 가능한 원인 확인 방법
프로그램이 실행되지 않음 링크 경로 오류 또는 이동 중 파일 누락 원본 링크 대상과 실제 폴더 내부 파일을 대조합니다.
설정이 초기화됨 사용자 데이터가 별도 위치에 있거나 일부 폴더만 이동됨 AppData와 프로그램 설정 저장 위치를 확인합니다.
실행 속도가 느려짐 보조 드라이브 속도 부족 HDD, 외장 드라이브, 네트워크 드라이브 사용 여부를 점검합니다.

예기치 못한 상황을 위한 원상 복구

보조 드라이브가 느리거나, 드라이브를 교체해야 하거나, 특정 프로그램이 링크 구조와 잘 맞지 않는다면 원래 상태로 되돌릴 수 있어야 합니다. 심볼릭 링크는 실제 데이터가 아니라 연결 통로이므로, 순서를 지키면 복구는 비교적 단순합니다.

복구 절차

  1. 관련 프로그램을 완전히 종료합니다.
  2. C드라이브의 원래 위치에 있는 심볼릭 링크 폴더를 삭제합니다.
  3. 보조 드라이브에 있는 실제 데이터 폴더를 복사합니다.
  4. 원래 C드라이브 경로에 붙여넣습니다.
  5. 실행 파일이 원래 위치에 존재하는지 확인합니다.
  6. 바탕화면 바로가기나 시작 메뉴에서 프로그램을 실행합니다.
  7. 설정과 작업 내역이 유지되는지 확인한 뒤 재부팅합니다.
“삭제해야 하는 것은 실제 데이터가 아니라 C드라이브에 남아 있는 연결 폴더입니다. 이 구분이 복구의 핵심입니다.”

실무적 시사점

심볼릭 링크 기반 이동은 저장 공간 관리에서 꽤 강력한 선택지입니다. 특히 대용량 게임, 영상 편집 도구, 개발 SDK처럼 재설치 비용이 큰 프로그램에 유용합니다. 다만 이 방식은 저장 위치를 바꾸는 것이지 프로그램 구조를 공식적으로 재설치하는 것은 아닙니다.

적합한 경우 신중해야 하는 경우
대용량 프로그램 폴더가 C드라이브를 압박하는 경우 백신, 드라이버, 시스템 서비스처럼 OS와 깊게 결합된 프로그램
재설치 시간이 오래 걸리거나 설정 복원이 번거로운 경우 외장 드라이브처럼 연결이 자주 끊기는 저장소를 대상 위치로 쓰는 경우
보조 SSD처럼 빠르고 안정적인 드라이브가 있는 경우 회사 보안 정책이나 관리 도구가 설치 경로 변경을 제한하는 환경
권장 운영 방식. 처음에는 중요도가 낮은 프로그램 하나로 테스트하고, 검증 절차가 익숙해진 뒤 대용량 프로그램에 적용하는 편이 안전합니다. 이동 전에는 중요한 설정과 작업 파일을 별도로 백업해 두는 것이 좋습니다.

마무리

C드라이브 용량 부족은 단순히 파일 몇 개를 지우는 방식으로는 반복되기 쉽습니다. 대용량 프로그램이 공간을 차지하고 있다면, FolderMove와 심볼릭 링크를 이용해 실제 데이터를 보조 드라이브로 옮기는 방법이 더 효과적일 수 있습니다.

핵심은 이동보다 검증입니다. 원본 경로, 대상 경로, 링크 상태, 프로그램 실행, 복구 가능성까지 확인하면 재설치 없이도 저장 공간을 안정적으로 확보할 수 있습니다. C드라이브가 빨간색 경고 막대를 보이기 시작했다면, 먼저 이동 가능한 대용량 프로그램부터 점검해 볼 만합니다.
