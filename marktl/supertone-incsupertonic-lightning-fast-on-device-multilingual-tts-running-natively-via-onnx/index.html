Research Report · Editorial Brief

Supertonic 3는 온디바이스 TTS의 기준을 어디까지 끌어올렸나.

99M 파라미터, 31개 언어, ONNX 기반 로컬 추론. Supertonic 3는 “음성 합성은 클라우드에서 해야 한다”는 전제를 작고 빠른 배포 모델로 다시 묻는다.

발행용 리서치 노트 작성일 2026년 5월 17일 주제 온디바이스 TTS
공유 문구

Supertonic 3는 브라우저, 모바일, 엣지 기기에서 직접 실행되는 다국어 TTS다. 이 글은 속도, 정확도, 모델 크기, 개발자 경험을 기준으로 제품적 의미를 정리한다.

초록. Supertonic은 Supertone Inc.가 공개한 온디바이스 다국어 텍스트 음성 변환 시스템이다. Supertonic 3는 31개 언어, 44.1kHz WAV 출력, 10개 표현 태그, Python부터 Rust와 Flutter까지 이어지는 다중 런타임 예제를 제공한다. 핵심 메시지는 단순하다. TTS를 클라우드 API가 아니라 로컬 추론 인프라로 다룰 수 있다는 것이다.

이 보고서는 원본 GitHub README의 기술 설명, 릴리스 노트, 성능 표, 배포 예제를 바탕으로 Supertonic 3의 강점과 실제 도입 시 검토해야 할 지점을 편집형 기사 구조로 재정리했다.

핵심 발견.

31

다국어 범위가 제품 단위로 확장됐다.

Supertonic 2의 5개 언어에서 Supertonic 3의 31개 언어로 넓어졌다. 한국어, 일본어, 영어, 프랑스어, 독일어, 스페인어 등 주요 언어가 포함된다.

99M

모델 크기가 엣지 배포에 맞춰져 있다.

공개 ONNX 에셋 기준 약 9,900만 파라미터다. 0.7B에서 2B급 공개 TTS 시스템보다 작다는 점이 콜드 스타트와 메모리 사용량에 직접 작용한다.

ONNX

배포 경로가 넓다.

Python, Node.js, WebGPU 브라우저, Java, C++, C#, Go, Swift, iOS, Rust, Flutter 예제를 제공한다. 연구 모델보다 SDK형 자산에 가깝다.

44.1kHz

출력 품질을 실사용 기준으로 제시한다.

외부 업샘플러 없이 44.1kHz 16비트 WAV 출력을 제공한다. 콘텐츠 제작, 리더 앱, 브라우저 확장에 바로 연결하기 쉬운 사양이다.

온디바이스 TTS의 진짜 가치는 “오프라인에서도 말한다”가 아니라, 개인정보와 지연시간을 제품 설계의 기본값으로 되돌린다는 데 있다.

왜 지금 중요한가.

TTS는 오랫동안 서버형 기능으로 취급됐다. 음성 품질과 모델 크기 때문에 클라우드 API를 호출하는 방식이 일반적이었다. 그러나 전자책 리더, 브라우저 확장, 모바일 접근성 도구, 로컬 LLM 인터페이스에서는 네트워크 왕복과 개인정보 전송이 모두 비용이다.

Supertonic 3의 주장은 이 지점에서 분명하다. 모델을 충분히 작게 만들고 ONNX 런타임으로 배포하면, TTS는 애플리케이션 안에 들어갈 수 있다. 사용자의 텍스트가 외부 서버로 나가지 않아도 되고, GPU가 없는 환경에서도 음성 합성을 수행할 수 있다.

입력 텍스트 자연어, 숫자, 통화, 전화번호, 약어, 표현 태그를 포함한 실제 문장.

그림 1. Supertonic 3가 지향하는 로컬 TTS 파이프라인.

근거 1. 작고 빠른 배포 모델.

Supertonic 3는 공개 ONNX 자산 기준 약 9,900만 파라미터를 갖는다. 원본 자료는 이 크기가 0.7B에서 2B급 공개 TTS 시스템보다 훨씬 작다고 설명한다. 작은 모델은 단순한 수치가 아니라 제품 비용이다. 다운로드 시간, 첫 실행 대기 시간, 모바일 저장공간, 브라우저 캐시, 메모리 압박이 모두 줄어든다.

99M Supertonic 3 공개 ONNX 에셋의 대략적 파라미터 수.
CPU GPU 없이 로컬 실행을 목표로 설계된 추론 경로.
Edge 라즈베리 파이, 전자책 단말기, 브라우저, 모바일 앱을 염두에 둔 배포.

특히 “웹페이지 전체를 1초 이내에 오디오로 변환할 수 있을 만큼 빠르다”는 원문 설명은 브라우저 확장이나 읽기 보조 도구에서 중요하다. 음성 합성은 품질만 좋아서는 부족하다. 사용자가 버튼을 누른 뒤 기다려야 하는 시간이 길어지면 제품 경험은 무너진다.

근거 2. 정확도 경쟁은 읽기 오류율에서 시작된다.

원본 자료는 Minimax-MLS-test 벤치마크를 기준으로 Supertonic 3와 VoxCPM2, OmniVoice, Qwen3-TTS, Supertonic 2를 비교한다. 모든 언어에서 최고 성능을 주장하는 구조는 아니다. 대신 “온디바이스 배포가 가능한 작은 모델임에도 더 큰 공개 TTS 모델과 경쟁 가능한 읽기 정확도 범위를 보인다”는 메시지에 가깝다.

언어 Supertonic 3 비교 관찰
영어 2.06 WER VoxCPM2 2.11, Qwen3-TTS 2.25, Supertonic 2 2.52와 비교해 경쟁력 있는 수치.
한국어 3.26 CER Supertonic 2의 3.65보다 낮고, Qwen3-TTS 4.07보다 낮다.
독일어 0.86 WER Qwen3-TTS 0.52보다는 높지만 VoxCPM2 0.85와 거의 같은 범위.
스페인어 1.13 WER VoxCPM2 1.34와 Supertonic 2 1.81보다 낮다.
우크라이나어 1.23 WER VoxCPM2 5.85보다 크게 낮지만 OmniVoice 0.71보다는 높다.

표 1. 원본 README의 언어별 WER 및 CER 표에서 일부 항목을 발췌해 해석했다. 값은 낮을수록 좋다.

읽는 법. WER은 단어 오류율, CER은 문자 오류율이다. 원본 표에서 아랍어, 힌디어, 일본어, 한국어처럼 별표가 붙은 언어는 CER로 평가됐다.

근거 3. 개발자 경험은 예제로 설계됐다.

Supertonic의 강점은 모델 파일만 던져주는 데서 끝나지 않는다는 점이다. Python 패키지는 첫 실행 시 Hugging Face에서 모델 에셋을 자동으로 다운로드할 수 있고, 저장소는 주요 런타임별 예제를 제공한다.

pip install supertonic from supertonic import TTS tts = TTS(auto_download=True) wav, duration = tts.synthesize( text="Supertonic is a lightning fast, on-device TTS system.", lang="en", voice_style=style, total_steps=8, speed=1.05, ) tts.save_audio(wav, "output.wav")

Node.js nodejs/ 서버 사이드 자바스크립트 환경에서 사용 가능.

지원 언어와 표현 태그.

Supertonic 3는 아랍어, 불가리아어, 크로아티아어, 체코어, 덴마크어, 네덜란드어, 영어, 에스토니아어, 핀란드어, 프랑스어, 독일어, 그리스어, 힌디어, 헝가리어, 인도네시아어, 이탈리아어, 일본어, 한국어, 라트비아어, 리투아니아어, 폴란드어, 포르투갈어, 루마니아어, 러시아어, 슬로바키아어, 슬로베니아어, 스페인어, 스웨덴어, 터키어, 우크라이나어, 베트남어를 지원한다.

언어를 모르는 입력에는 lang="na"를 사용할 수 있다. 또한 <laugh>, <breath>, <sigh> 같은 인라인 표현 태그를 통해 참조 오디오 없이도 일부 자연스러운 발화 뉘앙스를 추가할 수 있다.

  • 다국어 제품에서 언어별 모델을 따로 붙이는 부담을 줄인다.
  • 오프라인 리더, 접근성 도구, 브라우저 확장에 직접 통합하기 쉽다.
  • 감정 연기 전체를 대체한다기보다 짧은 표현 신호를 제공하는 방식이다.

실제 텍스트 처리에서 드러나는 차별점.

원본 자료는 금융 표현, 전화번호, 기술 단위처럼 TTS 시스템이 자주 틀리는 입력을 별도로 비교한다. 핵심은 음질보다 먼저 “읽을 수 있는가”다. 숫자와 약어를 잘못 읽으면 아무리 자연스러운 목소리라도 제품 신뢰도가 떨어진다.

범주 어려운 입력 제품적 의미
금융 표현 $5.2M, $450K, 통화 기호와 규모 단위. 뉴스, 리포트, 투자 문서를 읽을 때 오류가 크게 드러난다.
전화번호 지역번호, 하이픈, 내선 번호, ext. 표기. 호텔, 고객지원, 안내문 음성화에서 실사용성을 좌우한다.
기술 단위 2.3h, 30kph, 공학적 약어와 소수점. 매뉴얼, 제품 사양, 기술 문서를 읽을 때 중요하다.

이 비교는 독립 검증이 아니라 프로젝트 README에 포함된 자체 예시다. 따라서 절대적 성능 판정으로 보기보다는 Supertonic이 어떤 문제를 제품 차별점으로 보고 있는지 보여주는 자료로 해석하는 편이 적절하다.

도입 시사점.

Supertonic 3는 “가장 큰 TTS 모델”을 겨냥하지 않는다. 대신 배포 가능한 충분한 품질, 빠른 로컬 실행, 낮은 인프라 의존성이라는 조합을 제시한다. 이 조합은 다음 제품군에서 특히 강하다.

  • 웹페이지를 즉시 읽어주는 브라우저 확장.
  • 비행기 모드에서도 동작해야 하는 전자책 리더와 학습 앱.
  • 개인 문서를 외부 서버로 보내기 어려운 기업용 접근성 도구.
  • 로컬 LLM 챗봇의 음성 출력 레이어.
  • 라즈베리 파이 같은 저전력 기기에서 작동하는 안내 시스템.

반대로 극도로 세밀한 감정 연기, 배우급 음색 제어, 대규모 음성 클로닝, 스튜디오 후반 작업을 요구하는 환경이라면 별도의 평가가 필요하다. Supertonic 3의 강점은 고급 음성 제작 플랫폼 전체를 대체하는 것보다, “항상 켜져 있고 빠르게 반응하는 로컬 읽기 엔진”에 가깝다.

소스 노트.

  1. 주요 출처는 supertone-inc/supertonic GitHub 저장소의 README 내용이다.
  2. 모델 및 데모 참조는 Hugging Face Supertonic 3 모델 페이지와 Hugging Face Space 데모를 기준으로 정리했다.
  3. 성능 수치는 원본 README에 제시된 Minimax-MLS-test 기반 WER 및 CER 표를 요약한 것이다. 독립 재현 실험은 이 문서 범위에 포함하지 않았다.
  4. 원본 문서의 원격 이미지와 동영상은 게시용 단일 HTML의 안전한 배포를 위해 포함하지 않았다.
  5. 라이선스 관련 항목은 샘플 코드 MIT 라이선스, 모델 OpenRAIL-M 라이선스라는 원문 설명을 따랐다.

마지막 판단.

Supertonic 3의 흥미로운 점은 연구 모델의 데모가 아니라 제품에 넣을 수 있는 형태로 TTS를 제시한다는 데 있다. 클라우드 음성 합성이 여전히 강력한 선택지인 것은 맞다. 그러나 개인정보, 지연시간, 오프라인 동작, 배포 비용이 중요한 제품이라면 Supertonic 3는 검토할 만한 기준선을 만든다.
