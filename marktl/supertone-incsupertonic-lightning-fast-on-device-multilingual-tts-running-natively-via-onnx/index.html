Research Report · Editorial Brief

Supertonic 3는 온디바이스 TTS의 무게중심을 어디까지 옮길 수 있을까.

99M 파라미터, 31개 언어, ONNX 기반 로컬 추론, 44.1kHz 출력. Supertonic 3는 “클라우드 호출 없이도 실용적인 다국어 음성 합성이 가능한가”라는 질문에 제품 수준의 답을 제시한다.

  • 작성 기준일 2026년 5월 17일.
  • 원문 출처 supertone-inc/supertonic GitHub 저장소.
  • 형식 공개용 리서치 리포트.

공유용 한 줄 요약

Supertonic 3는 31개 언어를 지원하는 99M 파라미터 온디바이스 TTS로, 브라우저와 엣지 기기까지 겨냥한 공개 ONNX 음성 합성 모델이다.

온디바이스 AI 음성 합성의 다음 기준은 더 큰 모델이 아니라 더 작고 빠르며 배포 가능한 모델일 수 있다.
목차 초록 핵심 발견 왜 중요한가 성능 근거 배포와 런타임 자연어 처리 생태계 주의점 시사점 출처 노트
Abstract

Supertonic은 Supertone Inc.가 공개한 초고속 온디바이스 다국어 TTS 시스템이다. 핵심 주장은 명확하다. 텍스트 음성 변환을 클라우드 API에 의존하지 않고, 데스크톱, 모바일, 브라우저, 라즈베리 파이, 전자책 단말기 같은 제한된 환경에서도 실행할 수 있다는 것이다.

이 리포트는 GitHub 저장소에 공개된 제품 설명, 성능 표, SDK 지원 범위, 모델 버전 정보를 바탕으로 Supertonic 3의 의미를 정리한다. 결론적으로 Supertonic 3의 강점은 단일 지표의 압도적 우위라기보다, “충분한 품질”, “작은 모델”, “다국어”, “로컬 배포”, “여러 런타임 예제”를 한 번에 묶은 실행 가능성에 있다.

핵심 발견.

31 Supertonic 3가 지원하는 언어 수. v2의 5개 언어에서 크게 확장됐다.
99M 공개 ONNX 자산 기준 모델 파라미터 규모. 0.7B~2B급 공개 TTS 모델보다 작다.
44.1kHz 외부 업샘플러 없이 출력하는 16비트 WAV 오디오 품질.
ONNX Python, Node.js, WebGPU, Java, C++, C#, Go, Swift, iOS, Rust, Flutter 예제를 관통하는 배포 기반.
Supertonic 3의 전략은 “가장 큰 음성 모델”이 아니라 “가장 쉽게 제품 안에 넣을 수 있는 음성 모델”에 가깝다.
  • 클라우드 호출 없이 개인정보 보호와 지연 시간 문제를 동시에 줄인다.
  • 브라우저와 엣지 디바이스를 포함해 배포 가능한 표면적을 넓힌다.
  • 표현 태그와 자연어 정규화로 실제 문장 읽기 문제를 직접 겨냥한다.
  • 오픈 웨이트와 공개 예제로 실험에서 제품 통합까지의 간격을 줄인다.

왜 중요한가.

TTS는 오랫동안 클라우드 API와 대형 모델 중심으로 발전해 왔다. 품질을 높이기 쉬운 대신, 네트워크 지연, 비용, 개인정보, 오프라인 사용성, 플랫폼 종속성이라는 비용을 감수해야 했다. Supertonic 3는 이 균형점을 다시 묻는다.

저장소 설명에 따르면 이 모델은 웹페이지 전체를 1초 이내에 오디오로 변환할 수 있을 만큼 빠른 저지연 합성을 목표로 한다. 또한 입력 언어를 모를 때 `lang="na"`로 언어 비의존 처리를 요청할 수 있고, GPU 없이 CPU 기반 로컬 추론을 전제로 한다.

성능 근거.

Supertonic 3는 Minimax-MLS-test 벤치마크에서 읽기 정확도를 제시한다. 별표가 붙은 언어는 CER, 나머지는 WER을 사용한다. 값은 낮을수록 좋으며, 공개 표에 따르면 Supertonic 3는 일부 언어에서 최고는 아니지만 더 큰 모델군과 비교 가능한 범위를 유지한다.

배포와 런타임.

Supertonic의 제품적 매력은 지원 런타임의 폭에서 나온다. Python SDK로 빠르게 시작할 수 있고, ONNX 런타임을 바탕으로 서버, 브라우저, 네이티브 앱, 모바일 앱까지 이어진다.

브라우저와 WebGPU.

onnxruntime-web 기반 클라이언트 추론을 통해 네트워크 호출 없는 웹 TTS 경험을 겨냥한다.

엣지 디바이스.

라즈베리 파이와 전자책 리더기 같은 제한된 하드웨어에서도 실행 가능하다는 점을 전면에 내세운다.

멀티 언어 SDK.

Python, Node.js, Java, C++, C#, Go, Swift, iOS, Rust, Flutter 예제가 공개되어 있다.

Python 빠른 시작.

pip install supertonic

from supertonic import TTS

tts = TTS(auto_download=True)

wav, duration = tts.synthesize(
    text="Supertonic is a lightning fast, on-device TTS system.",
    lang="en",
    voice_style=style,
    total_steps=8,
    speed=1.05,
)

tts.save_audio(wav, "output.wav")
운영 환경에서는 모델 에셋 다운로드, Git LFS, ONNX 런타임 C 라이브러리, JDK, .NET 런타임 등 플랫폼별 전제 조건을 미리 고정하는 것이 중요하다.

자연어 텍스트 처리.

저장소는 Supertonic이 자연스러운 산문, 구두점, 약어, 고유 명사, 단위, 통화, 전화번호 같은 실제 입력을 처리하도록 설계됐다고 설명한다. 이는 TTS 품질을 단순히 음색의 자연스러움으로만 보지 않는다는 뜻이다. 제품에서는 “무엇을 어떻게 읽는가”가 사용자 신뢰를 좌우한다.

범주 읽기 난점 Supertonic 주장
금융 표현 5.2M, $450K, 통화 기호와 규모 단위. 사전 처리 없이 올바른 통화 및 수량 발음을 목표로 한다.
전화번호 지역번호, 하이픈, ext., 내선 번호. 숫자를 문맥에 맞게 분리해 읽는 처리를 강조한다.
기술 단위 2.3h, 30kph 같은 약식 단위. 공학 및 제품 설명에서 요구되는 단위 발음을 다룬다.
음성 합성의 실전 품질은 목소리의 매끄러움만이 아니라 숫자, 단위, 약어를 틀리지 않는 능력에서 드러난다.

생태계와 사용 사례.

Supertonic으로 제작됐거나 연동된 프로젝트 목록은 이 모델이 단순 연구 공개물보다 배포 지향 자산에 가깝다는 점을 보여준다. 웹페이지 읽기 확장 프로그램, iOS 전자책 리더, 브라우저 음성 챗봇, Kotlin 멀티플랫폼 SDK, MNN 기반 경량 라이브러리, Transformers.js 지원 등이 언급되어 있다.

주의해서 읽어야 할 지점.

이 리포트는 원 저장소에 공개된 설명과 표를 바탕으로 한다. 따라서 Supertonic 3의 독립 재현 평가, 실제 기기별 지연 시간, 장문 안정성, 한국어 문장 처리의 세부 오류 유형, 상업 환경에서의 라이선스 검토는 별도로 확인해야 한다.

  • 성능 수치는 벤치마크와 측정 조건에 의존한다.
  • WER과 CER이 낮아도 음색 선호도, 감정 표현, 장문 내구성은 별도 평가가 필요하다.
  • OpenRAIL-M 모델 라이선스는 사용 목적과 배포 방식에 맞춰 검토해야 한다.
  • 브라우저, 모바일, 엣지 배포는 런타임 크기와 초기 로딩 경험까지 함께 봐야 한다.

시사점.

Supertonic 3가 던지는 메시지는 단순하다. 음성 합성은 더 이상 서버의 전유물이 아니다. 충분히 작은 모델, 표준화된 ONNX 배포, 폭넓은 런타임 예제, 다국어 지원이 결합되면 TTS는 브라우저 확장, 전자책 단말기, 모바일 앱, 로컬 생산성 도구 안으로 직접 들어갈 수 있다.

앞으로의 경쟁은 “누가 가장 인간 같은 목소리를 내는가”만이 아니라 “누가 가장 적은 의존성으로, 가장 많은 환경에서, 가장 안정적으로 읽어주는가”로 확장될 가능성이 크다. Supertonic 3는 그 변화의 꽤 구체적인 사례다.
