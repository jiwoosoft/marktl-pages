Abstract

꺼진 PC도 네트워크 신호로 깨울 수 있습니다

WOL, Wake-On-LAN은 전원이 꺼진 PC의 랜카드가 대기 전력을 유지하다가 매직 패킷을 받으면 메인보드에 부팅 신호를 전달하는 기능입니다. 성공 조건은 단순합니다. 유선 랜 연결, 랜카드 전원 관리 허용, 빠른 시작 해제, 라우터의 대상 PC 등록, BIOS의 PCI-E 전원 켜기 설정이 맞아야 합니다.

Key Takeaway

실패 원인은 대부분 네 곳에 있습니다

WOL이 되지 않을 때는 대개 빠른 시작, ErP, 무선 연결, 멀티탭 전원 차단 중 하나가 원인입니다. 외부 접속 테스트는 집 와이파이를 끄고 LTE나 5G에서 실행해야 실제 환경을 검증할 수 있습니다.

외출 중 집 PC에 중요한 파일이 있거나, 퇴근길에 고사양 게임과 원격 제어 환경을 미리 켜두고 싶을 때가 있습니다. 원격 제어 앱은 PC가 켜져 있어야 의미가 있으므로, 전원이 꺼진 상태를 해결하는 첫 단추가 바로 WOL입니다.

WOL은 공유기가 내 대신 전원 버튼을 누르는 기술이 아니라, 랜카드와 메인보드가 약속된 기상 신호에 반응하도록 만드는 설정 묶음입니다.

핵심 발견

유선 연결이 기준입니다

꺼진 PC는 무선 연결을 안정적으로 유지하기 어렵습니다. 데스크톱과 라우터는 유선 랜 케이블로 직접 연결하는 것이 가장 확실합니다.

윈도우와 BIOS가 함께 열려야 합니다

랜카드 설정만 켜도 부족합니다. BIOS의 ErP와 PCI-E 전원 관련 옵션까지 맞아야 매직 패킷이 전원 신호로 이어집니다.

외부 테스트가 진짜 검증입니다

같은 집 와이파이 안에서 성공해도 외부 원격 부팅이 검증된 것은 아닙니다. 스마트폰을 LTE나 5G로 바꿔 테스트해야 합니다.

WOL의 개념과 사전 준비

WOL은 Wake-On-LAN의 약자로, 네트워크를 통해 PC를 깨우는 기능입니다. 컴퓨터를 시스템 종료해도 메인보드와 랜카드는 아주 작은 대기 전력을 사용하며 신호를 기다릴 수 있습니다. 이때 스마트폰 앱이나 라우터가 매직 패킷이라는 특수 신호를 보내면 랜카드가 이를 감지하고 메인보드에 부팅을 요청합니다.

1스마트폰 앱에서 전원 켜기 요청
2라우터가 매직 패킷 전송
3랜카드가 대기 전력으로 신호 수신
4메인보드가 PCI-E 기상 신호 처리
5PC 전원 켜짐

사전 체크리스트

  • 유선 랜선 연결 공유기와 PC가 유선 랜케이블로 연결되어 있어야 합니다.
  • WOL 지원 랜카드와 메인보드 최근 10년 이내 데스크톱 대부분은 지원하지만 설정이 꺼져 있을 수 있습니다.
  • WOL 지원 공유기 ipTIME, ASUS 등 주요 공유기는 관련 기능을 제공합니다. 이 글은 ASUS 라우터를 기준으로 설명합니다.

윈도우 네트워크 어댑터와 랜카드 설정

윈도우에서는 유선 랜카드가 절전 상태에서도 신호를 받을 수 있도록 전원 관리와 고급 속성을 조정해야 합니다. 기본값은 절전 우선으로 되어 있는 경우가 많아 WOL 신호를 막을 수 있습니다.

설정 경로

  1. 작업 표시줄 검색에서 네트워크 연결을 입력합니다.
  2. 네트워크 연결 보기를 실행합니다.
  3. 현재 유선으로 연결된 이더넷 어댑터를 우클릭합니다.
  4. 속성을 열고 구성 버튼을 누릅니다.

전원 관리 탭에서 확인할 항목

항목 권장 상태 이유
전원을 절약하기 위해 컴퓨터가 이 장치를 끌 수 있음 체크 운영체제가 장치 전원 관리를 수행할 수 있게 합니다.
이 장치를 사용하여 컴퓨터의 대기 모드를 종료할 수 있음 체크 랜카드가 절전 또는 종료 상태의 깨우기 신호를 전달할 수 있게 합니다.
매직 패킷에서만 컴퓨터의 대기 모드를 종료할 수 있음 체크 불필요한 네트워크 신호가 아니라 WOL 신호에만 반응하도록 제한합니다.

고급 탭에서 켤 항목

매직 패킷 웨이크 온 또는 Wake on Magic Packet 항목을 찾아 값을 사용 또는 Enabled로 바꿉니다. 패턴 일치 웨이크 온 또는 Wake on Pattern Match가 있다면 함께 사용으로 설정합니다.

윈도우 전원 옵션과 빠른 시작 끄기

윈도우 10과 11의 빠른 시작은 부팅 속도를 높이기 위해 종료 상태를 일반적인 완전 종료와 다르게 처리합니다. 이 기능이 WOL 신호 수신을 방해하는 경우가 많으므로 원격 부팅 목적이라면 끄는 편이 안전합니다.

빠른 시작 해제 순서

  1. 작업 표시줄 검색에서 전원 관리 옵션을 입력합니다.
  2. 전원 관리 옵션 편집을 실행합니다.
  3. 상단 경로에서 전원 옵션으로 이동합니다.
  4. 왼쪽의 전원 단추 작동 설정을 선택합니다.
  5. 현재 사용할 수 없는 설정 변경을 누릅니다.
  6. 빠른 시작 켜기 체크를 해제합니다.
  7. 변경 내용 저장을 누릅니다.

ASUS 라우터 WOL 대상 PC 등록

PC 쪽 설정이 끝났다면 라우터가 어느 장치를 깨워야 하는지 알아야 합니다. ASUS 라우터에서는 웹 관리자 페이지의 네트워크 도구 메뉴에서 WOL 대상 장치를 등록할 수 있습니다.

기본 게이트웨이 확인

윈도우 설정의 네트워크 및 인터넷에서 이더넷 속성을 열고 IPv4 기본 게이트웨이를 확인합니다. 보통 192.168.50.1 같은 형식이며, 이 주소가 라우터 관리자 페이지로 들어가는 주소입니다.

라우터 관리자 페이지 등록 절차

  1. 브라우저 주소창에 기본 게이트웨이 주소를 입력합니다.
  2. ASUS 라우터 관리자 계정으로 로그인합니다.
  3. 왼쪽 메뉴에서 네트워크 도구를 엽니다.
  4. 상단 탭에서 Wake on LAN을 선택합니다.
  5. 드롭다운에서 유선 연결된 대상 PC를 선택합니다.
  6. 추가 버튼을 누르고 적용으로 저장합니다.
라우터 단계 확인해야 할 값 실패 시 의심 지점
관리자 페이지 접속 IPv4 기본 게이트웨이 잘못된 네트워크에 연결됨
대상 PC 선택 PC 이름 또는 MAC 주소 PC가 유선으로 연결되지 않음
WOL 적용 저장된 클라이언트 목록 적용 버튼 누락 또는 라우터 재부팅 필요

메인보드 BIOS 전원 관리 설정

윈도우 설정이 맞아도 BIOS가 대기 전력을 차단하면 WOL은 작동하지 않습니다. ASUS 메인보드 기준으로는 BIOS의 고급 모드에서 APM, Advanced Power Management 메뉴를 확인합니다.

BIOS 진입과 이동

  1. PC 전원을 켜자마자 F2 또는 Del 키를 반복해서 누릅니다.
  2. 간단 모드가 열리면 F7로 고급 모드로 전환합니다.
  3. Advanced 탭으로 이동합니다.
  4. APM 메뉴를 엽니다.

핵심 BIOS 옵션

옵션 권장값 의미
ErP Ready Disabled 대기 전력을 극단적으로 줄이는 기능을 꺼서 랜카드가 신호를 받을 수 있게 합니다.
Power On By PCI-E Enabled 랜카드나 PCI-E 장치에서 들어오는 기상 신호로 PC 전원을 켤 수 있게 합니다.
Wake on LAN 또는 PCIE Wake Enabled 제조사와 BIOS 버전에 따라 이름이 다를 수 있는 WOL 관련 옵션입니다.

설정 변경 후에는 F10을 눌러 저장하고 재시작합니다. 최신 BIOS에서는 관련 기능이 기본 활성화되어 옵션 자체가 보이지 않을 수도 있습니다.

스마트폰으로 외부 원격 부팅 실행

PC, 라우터, BIOS 설정이 끝나면 ASUS Router 앱에서 원격 부팅을 실행할 수 있습니다. 앱은 라우터에 등록된 대상 PC를 보여주고, 사용자는 해당 장치의 전원 켜기 버튼을 누르면 됩니다.

ASUS Router 앱 사용 순서

  1. iPhone App Store 또는 Android Play Store에서 ASUS Router 앱을 설치합니다.
  2. 라우터 관리자 계정으로 로그인합니다.
  3. 하단 탭에서 설정을 엽니다.
  4. 네트워크 도구로 이동합니다.
  5. Wake-On-LAN 메뉴를 엽니다.
  6. 등록된 대상 PC 옆의 전원 또는 번개 아이콘을 누릅니다.
최종 테스트는 반드시 스마트폰 와이파이를 끄고 LTE나 5G에서 실행해야 합니다. 그래야 외부 원격 부팅이라는 실제 목적을 검증할 수 있습니다.

주의사항과 실패 방지 체크

WOL은 한 번 설정하면 편리하지만, 전원과 네트워크의 작은 변화에도 실패할 수 있습니다. 특히 멀티탭 전원 차단, 무선 연결, 강제 종료, 빠른 시작은 반복적으로 문제를 만드는 지점입니다.

  • 멀티탭 전원 스위치를 끄지 않습니다 메인보드와 랜카드에는 대기 전력이 공급되어야 합니다.
  • 유선 랜 케이블을 유지합니다 Wi-Fi 연결만으로는 종료 상태의 WOL을 안정적으로 기대하기 어렵습니다.
  • 강제 종료를 피합니다 전원 버튼 장시간 누름이나 정전 후에는 WOL 대기 상태가 꼬일 수 있습니다.
  • 외출 전 셀룰러 테스트를 합니다 LTE 또는 5G에서 앱으로 PC가 켜지는지 확인합니다.

문제가 생겼을 때 우선순위

1순위는 빠른 시작 해제, 2순위는 BIOS의 ErP 비활성화, 3순위는 라우터의 대상 PC 등록 상태입니다. 이 세 곳을 다시 확인하면 대부분의 WOL 실패를 좁힐 수 있습니다.

실무 적용 시사점

WOL은 단순한 편의 기능을 넘어 원격근무, 개인 서버 운영, 고성능 데스크톱 원격 접속, 게임 스트리밍 환경에서 전력과 접근성을 함께 잡는 방법입니다. 항상 켜두는 방식보다 전력 낭비가 적고, 필요할 때만 PC를 켤 수 있다는 점에서 개인 사용자에게 특히 유용합니다.

다만 보안 관점에서는 라우터 관리자 계정과 원격 접근 설정을 함께 점검해야 합니다. 강한 관리자 암호를 사용하고, 라우터 펌웨어를 최신 상태로 유지하며, 불필요한 외부 관리 기능은 꺼두는 것이 좋습니다.

마지막 정리

WOL 설정의 핵심은 하나의 스위치가 아니라 연결된 설정들의 연쇄입니다. 랜카드가 신호를 받을 수 있어야 하고, 윈도우가 이를 막지 않아야 하며, BIOS가 대기 전력을 허용해야 하고, 라우터가 정확한 PC에 매직 패킷을 보내야 합니다.

이 네 구간이 맞으면 스마트폰 하나로 외부에서 집 PC를 켤 수 있습니다. 원격 제어 앱을 쓰는 사람이라면 WOL은 선택 기능이 아니라 원격 환경의 시작점에 가깝습니다.

공유 문구. 꺼진 PC 때문에 원격 접속이 막힌 적 있다면, WOL 설정으로 스마트폰 원격 부팅 환경을 먼저 만들어 보세요.
